# CEP-CC Roadmap

## Stage 1: Proto-Symbol Emergence

Question:

> Does continuous communication cluster into stable reusable trajectory prototypes under compression?

Build:

- continuous object-selection environment.
- Speaker/Listener communication loop.
- compression sweep over lambda.
- clustering metrics.

Expected artifact:

- `CEP_CC_V0_1_PROTO_SYMBOL_EMERGENCE_YYYY-MM-DD.md`

Decision:

- continue only if compression creates reusable structure without destroying task success.

## Stage 2: Semantic Stability

Question:

> Do trajectory prototypes encode task-relevant invariants rather than nuisance statistics?

Build:

- environment reparameterization tests.
- translation, rotation, attribute-distribution shift, object identity reshuffle.
- probes from communication to latent invariant.

Expected artifact:

- `CEP_CC_V0_2_SEMANTIC_STABILITY_YYYY-MM-DD.md`

Decision:

- continue only if clusters maintain meaning under nuisance transformations.

## Stage 3: Local Compositionality

Question:

> Does the communication trajectory contain reusable local segments whose intervention effects are factor-specific?

Build:

- segment slicing.
- segment swap intervention.
- targeted/off-target effect metric.
- task family with relation composition.

Expected artifact:

- `CEP_CC_V0_3_COMPOSITIONALITY_YYYY-MM-DD.md`

Decision:

- call it language-like only if prototype reuse, semantic stability, and local compositionality all appear.

## Stage 4: Protocol As Cognitive Necessity

Question:

> Does communication become an external working-memory or shared-world-model loop?

Build:

- weak short-term memory agents.
- delayed evidence tasks.
- multi-view complementarity.
- value coordination tasks.

Expected artifact:

- `CEP_CC_V0_4_PROTOCOL_MEMORY_PRESSURE_YYYY-MM-DD.md`

Decision:

- move from emergent protocol to minimal language-like cognition only if communication is necessary for long-horizon control.

## Stage 5: Paper Packaging

Question:

> What is the narrow defensible claim?

Likely claim if stages pass:

> Discrete reusable protocol structure can emerge from continuous communication under compression and coordination pressure without tokenization or next-token prediction.

Likely negative claim if stages fail:

> Compression alone is insufficient; semantic stability or compositionality requires additional memory, multi-view, or value-coordination pressure.
