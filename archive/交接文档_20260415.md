# Unified-SEL 项目交接文档
## 一、概述
### 1.1 项目背景
Unified-SEL是一个持续学习研究项目，核心目标是验证「惊讶驱动的结构生灭机制在无任务边界信号下的表现优于需要显式任务边界的方法（如EWC）」。项目当前处于A1阶段：在toy problem（4维线性可分二分类任务）上验证核心假设，目标是让Unified-SEL的平均准确率超过EWC，遗忘率低于EWC，且统计显著（p<0.05）。

### 1.2 交接信息
- 交接时间：2026年4月15日
- 项目当前阶段：Phase 13
- 当前工作进展：已完成预测分歧路由策略的实现与15种子正式对比实验

## 二、已完成核心工作
### 2.1 Oracle路由上限测试
- 目的：测量在完美路由情况下（任务0输入全部路由到快照专家，任务1输入全部路由到当前模型）的理论性能上限，判断瓶颈是路由质量还是学习质量
- 实现：创建`F:\unified-sel\experiments\oracle_routing_bound.py`
- 结果：Oracle上限平均准确率=0.7863，远高于EWC基线的0.5005，证明瓶颈是路由质量而非学习质量

### 2.2 Surprise信号质量分析
- 目的：诊断原有惊讶门控路由失效的原因
- 实现：创建`F:\unified-sel\experiments\analyze_surprise_routing.py`，测量任务0和任务1测试输入的惊讶值分布
- 结果：任务0平均惊讶值=0.3249，任务1平均惊讶值=0.3192，两者差距仅0.0057，几乎完全重叠，惊讶信号无任务区分能力

### 2.3 预测分歧路由实现
- 目的：替换失效的惊讶门控路由，采用更有效的路由策略
- 实现：修改`F:\unified-sel\core\learner.py`的`_ensemble_predict`方法，当快照专家与当前模型预测类别不一致时，选择置信度更高的结果
- 逻辑说明：原有`_snapshot_surprise_threshold`参数被重新定义为置信度比率阈值：`snap_conf > current_conf * threshold`时使用快照专家结果

### 2.4 路由参数扫描
- 目的：寻找最优的置信度比率阈值
- 实现：创建`F:\unified-sel\experiments\disagreement_routing_scan.py`，测试阈值0.0、0.5、1.0、1.5、2.0
- 结果：最优阈值=1.0（当分歧时直接选择置信度更高的专家），5种子测试平均准确率=0.5035，略高于EWC基线的0.5005

### 2.5 15种子正式对比实验
- 目的：与EWC基线进行正式统计对比，验证预测分歧路由的效果
- 实现：创建`F:\unified-sel\experiments\A1_fix5_disagreement_15seed.py`，包含排列检验、bootstrap置信区间、Cohen's d效应量计算
- 结果：预测分歧路由平均准确率=0.4996 vs EWC=0.5005，差异无统计显著性（p=0.8982），但任务0准确率相比基线提升21%（0.3568 vs 0.2956）

## 三、关键技术发现
1. **瓶颈定位**：当前系统性能瓶颈是路由质量而非学习质量，理论上限可达0.7863，远高于现有性能
2. **惊讶信号失效**：原有惊讶信号无法区分任务0和任务1输入，差距仅0.0057，几乎完全重叠
3. **预测分歧路由效果**：新路由策略在保持平均准确率与EWC相当的情况下，显著提升了旧任务（任务0）的准确率（+21%）
4. **性能差距**：当前预测分歧路由距离Oracle上限还有0.2867的提升空间，核心是需要更好的路由信号

## 六、当前问题与风险
1. **路由信号不足**：现有预测分歧路由仅利用了置信度信息，没有利用数据分布特征，性能提升有限
2. **任务0准确率偏低**：当前任务0准确率仅0.3568，远低于EWC的0.9070，需要更好的路由策略将旧任务输入路由到快照专家
3. **统计显著性不足**：当前平均准确率与EWC相当，尚未达到击败EWC的核心目标
4. **参数复用问题**：`_snapshot_surprise_threshold`参数被重新定义为置信度比率阈值，含义与原名称不符，可能造成混淆

## 七、下一步工作计划（按优先级）
### 高优先级
1. **用TopoMem的Wasserstein漂移替代surprise信号**：引入更强的任务区分信号，提高路由准确率
2. **修复参数命名问题**：将`_snapshot_surprise_threshold`重命名为更符合当前含义的名称，如`_snapshot_confidence_ratio_threshold`
3. **优化预测分歧路由逻辑**：结合分布漂移信号与置信度信息，进一步提升路由准确率

### 中优先级
1. **引入SEL-Lab的任务签名特征**：使用6维任务签名特征作为路由信号，提升任务区分能力
2. **扩展到5任务序列**：验证路由策略在更多任务场景下的有效性
3. **实现物理引导的动态阈值**：使用SDAS的r=lambda/lambda_c机制动态调整路由阈值，减少手动调参

### 低优先级
1. **替换真实LLM求解器**：在capability track上验证路由策略的有效性
2. **实现贝叶斯惊讶信号**：用后验分布变化替代启发式惊讶计算，提升信号精度
