## 二、核心发现（2026-04-06）

### 2.1 H1/H2 物理含义（最重要结论）

> **H1 = embedding 空间几何完整性签名（geometric integrity signature）**
> **H2 = 跨域边界复杂度指标（inter-domain boundary complexity）**

通过 A1（域分离敏感性）、A2（域数量敏感性）、A3（真实项目）、P0（随机对照）的系统性验证，确立了以下结论：

| 发现 | 结论 | 统计显著性 |
|------|------|-----------|
| A1: 分离 vs 混合域 H2 差异 | H2 编码域边界复杂度 | p<0.001, d=0.93 |
| A2: H2 随域数量单调增加 | H2 ∝ 域边界数 | Spearman r=0.926 |
| A3: 真实项目 H2 profile 各异 | 不同项目有不同 H2 指纹 | deer-flow/hermes/cognee 各不相同 |
| P0: 随机打乱后 H2 不变 | H2 是几何现象，非随机波动 | REAL vs SHUFFLED 无显著差异 |
| P1: H2 计算开销 <12ms | 计算开销可接受 | at n=100, overhead=11.9ms |

**H2 的物理含义（精确定义）**：
- H2 测量 embedding 点云中 **2D cavities（2维空洞）**的数量
- 每个 cavity = 一个潜在的"跨域边界区域"
- 当域数量增加时，域边界数量增加 → 更多 H2 cavities
- **H2/H1 ratio** 是敏感的域混合指标（比原始 H2 count 更稳定）

### 2.2 碎片化问题（H0/n = 1.000）

**发现**：在所有维度（包括 UMAP 降维后），VR filtration 都产生 H0/n = 1.000（每个点完全孤立）。

**根本原因**：
- Cosine metric 在单位球上，所有点的 birth = 0（半径 0 时每个点就存在）
- H0 birth 不编码簇结构，而是编码点间的filtration radius
- **真正有信息量的是 H0 persistence (death - birth)**

**对检索的影响**：
- H0 TDA 在 cosine metric + 384D 空间里不能直接提供检索增益
- 所有方法（PureVec / TopoMem-Hybrid / kNN）在 deer-flow 语料上打平
- Cosine similarity 本身已经足够区分这些域

### 2.3 H0 瓶颈（非 bug，是方法局限）

**VR filtration 的根本限制**：
- VR  filtration 在高维单位球上，每个点在 birth=0 就出现
- 点间距离范围很窄（cosine similarity 0.3-0.8），导致 filtration 没有语义分辨率
- 这不是代码 bug，是 metric × dimension 的数学本质

### 2.4 UMAP 降维实验（A4，部分完成）

**结论**：UMAP(384D → dD) 没有解决碎片化问题：
- H0/n 保持在 1.000（所有维度都碎片化）
- UMAP(30D) 实际上降低了 H2 的域敏感性（d 从 0.93 降到 0.46）
- UMAP 破坏了高维空间的域分离信号

**环境障碍**：torch 无法在 F: venv 加载（DLL 依赖），完整 UMAP + 真实编码实验需要稳定 GPU 环境。

### 3.3 TopoMem 最终定位

```
TopoMem = 多维拓扑健康监控系统
├── H0: 功能模块划分（via ChromaDB clustering）
├── H1: 嵌入空间几何完整性指标（geometric quality meter）
│   └── 用途：监控 embedding 漂移、灾难性遗忘的早期信号
└── H2: 跨域边界复杂度指标（domain boundary complexity）
    └── 用途：检测语义域侵入、多域混合程度
```

**重要**：H1/H2 不改进检索精度，但提供检索系统无法提供的**几何健康信号**。
